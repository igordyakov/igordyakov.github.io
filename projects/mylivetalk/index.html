MyLiveTalk for Windows

Wireframes

The first stage is about main structures and interactions which will allow Konstantin to start the development process. On the next step I’ll focus on visual style while Konstantin will be busy with building the fundament of the app. Using such parallel approach will allow us to reduce time and focus on things that really matter on each single step.

Login

At the very beginning user launches the application and the login screen appears. It has branding area at the top, obvious controls under it and some service “links” that will lead to the web part of the service.

It has four states:

0.0 Launch

0.1 User fills the fields

0.2 Validation if something goes wrong

0.3 Everything is OK and app is loading. Usually it will take only a few seconds but this state is required to give user feedback that “everything is OK, please wait”.

After the app is loaded the main window appears.

Contacts and Home Section

Let’s focus on sidebar with contacts at first. In sidebar we display only actual contacts and groups that user has chosen from Home section or had a conversation/notification from them recently.

Home section contains all contacts and groups with live search.

User opens Home section.

User uses live search to find desired contact/group.

When user chooses any contact/group from the list, it moves to the sidebar and appropriate conversation opens.

This approach will allow us to keep things clean and simple even for companies with tons of people and groups.

Conversation

The most common thing is obviously text messages. Bubbles are being grouped when user is sending chain of messages.

User can choose file through Explorer by clicking on button on the left side of the input field or just drag and drop it into the conversation area.

Kevin is sending the file with prices by drag-and-drop in the conversation area and finishing his text message to Anthony while this file is loading.

When Kevin clicks Send button, the file and message appear in the conversation area.

Text message wasn’t sent due to temporary network problems so he should click on it to retry.

There are some user details on the top of the conversation area (userpic, status, position, email etc.).

By clicking on search icon a string with local search appears. It works obviously, like Ctrl+F in browser or document.

By clicking on grid icon a little popup with grid of sent or received files appears. It’s an additional but useful feature that is somewise missed in most chat apps with file transfer functionality.

Status and User Settings

By clicking on the User Area in the sidebar context menu with status options appears. We may place here some service related things (like settings, help, about etc) as well.

User can set the custom status by clicking on appropriate item in context menu and typing the status message.

Conclusion

Initially chat apps is very conservative area where users have deep-seated habits and it’s better to keep them as they are. We have combined the best practices and patterns in a lightweight and clean UI where additional functionality can be added painlessly in future.
